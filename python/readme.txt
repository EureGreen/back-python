# Система разграничения прав доступа

## Концепция
Система основана на комбинации ролей и прямых разрешений (RBAC + ACL).

### Основные компоненты:
1. **Пользователи (Users)** - основные субъекты системы
2. **Роли (Roles)** - группы с набором прав
3. **Разрешения (Permissions)** - конкретные права на действия
4. **Ресурсы (Resources)** - объекты, к которым применяются права
5. **Правила доступа (Access Rules)** - связь ролей/пользователей с разрешениями на ресурсы

### Иерархия прав:
- Системные разрешения (глобальные)
- Разрешения на уровне ролей
- Индивидуальные разрешения пользователей
- Запрещающие правила (имеют приоритет над разрешающими)

### Типы ресурсов:
- Пользователи (users)
- Роли (roles)
- Разрешения (permissions)
- Бизнес-объекты (products, orders, etc.)

### Операции:
- create (создание)
- read (чтение)
- update (обновление)
- delete (удаление)
- manage (управление - все операции)
